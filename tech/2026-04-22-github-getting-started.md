# GitHub入门：从零到建立第一个笔记仓库

> 2026-04-22 | Tech

## 今日收获

### 1. GitHub的本质
GitHub不是"代码网盘"，而是全球软件协作的操作系统。
它解决三个底层问题：版本追溯、并行协作、信任分发。

### 2. 公开 ≠ 开源
开源需要附带License(许可证)，明确授予使用权。
没有License的公开代码，法律上默认"保留所有权利"。

### 3. 工具选择的第一性原理
解决问题优先选成熟工具(如Notion)，不要为了练手用半成品程序。
客户拜访记录用Notion，GitHub留给真正的学习输出。

### 4. README的价值
README是仓库的门脸——能否3秒让人理解"这是什么"。
写好README是产品经理思维的体现。

### 5. 工具链的搭建
- **GitHub**：云端仓库、版本管理、公开履历
- **GitHub Desktop**：本地与云端的同步桥梁
- **VS Code**：本地编辑器，未来也能写代码
- **三者协同**：在VS Code改 → GitHub Desktop提交 → 推到GitHub

## 关键概念速查

| 概念 | 一句话解释 |
|------|----------|
| Repository | 仓库，一个项目的容器 |
| Commit | 提交一次改动，有完整历史记录 |
| Push | 把本地改动推到云端 |
| Clone | 把云端仓库下载到本地 |
| README | 仓库的门脸说明书，自动渲染 |
| License | 授权别人使用的法律文件 |
