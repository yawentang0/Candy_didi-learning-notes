# Git / GitHub / GitHub Desktop / VS Code：四个工具的架构关系

> 2026-04-22 | Tech | 架构理解

## 问题起点

完成第一次Push之后，我意识到自己只是"跟着做"，并不理解这些工具之间到底是怎么连起来的。
特别是：**VS Code是怎么连到GitHub的？**

答案是——**VS Code根本没直接连GitHub**。中间隔着两层。

## 核心认知：真正的连接点只有一个

**所有工具的共同战场是本地的那个文件夹。**

`/Users/candytang/Documents/GitHub/learning-notes/`

这是一个普通的操作系统文件夹，但多了一个隐藏子目录 `.git/`——
这就是Git的"魔法"所在，记录了这个文件夹的所有历史版本。

VS Code只是在编辑这个文件夹里的文件，它完全不需要"连"GitHub。

## 四个角色的MECE拆分

| 角色 | 位置 | 职责 | 类比 |
|------|------|------|------|
| **GitHub** | 云端 | 存储、展示、协作 | 中央档案馆 |
| **Git** | 本地电脑 | 记录版本变化 | 24小时秘书 |
| **GitHub Desktop** | 本地电脑 | Git的图形操作界面 | 收发快递员 |
| **VS Code** | 本地电脑 | 编辑文件 | 笔和纸 |

**关键洞察**：Git才是真正的主角，GitHub Desktop只是它的"遥控器"。
